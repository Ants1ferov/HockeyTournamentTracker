# План: правила очков, таблица, пакет

## 1. Просмотр и редактирование правил подсчёта очков у существующего турнира

**Проблема:** На экране деталей турнира нет ни отображения текущих правил, ни возможности их изменить.

**Подход:**

- **Отображение на экране деталей:** В [TournamentDetailsPage.xaml](Presentation/Views/TournamentDetailsPage.xaml) после блока «Статус» добавить секцию «Правила начисления очков»: 6 подписей с текущими значениями из `Tournament.Rules` (В, ВО, ВБ, П, ПО, ПБ — например, «В: 3», «ВО: 2», …). Данные привязать к свойствам ViewModel.
- **ViewModel:** В [TournamentDetailsViewModel.cs](Presentation/ViewModels/TournamentDetailsViewModel.cs) добавить шесть свойств только для чтения (например, `PointsRegWinText`, `PointsOtWinText`, …), заполняемых при загрузке турнира из `Tournament.Rules`, и команду/метод перехода на страницу редактирования правил.
- **Страница редактирования правил:** Новая страница (например, `TournamentRulesEditPage` + `TournamentRulesEditViewModel`), принимающая `TournamentId` в query. Загрузка турнира по Id, форма с 6 полями ввода (как на [TournamentEditPage](Presentation/Views/TournamentEditPage.xaml) для правил), при сохранении — создание нового объекта `TournamentRules` с введёнными значениями, присвоение `tournament.Rules = newRules`, вызов `_tournamentRepository.SaveAsync(tournament)` и возврат назад. Регистрация страницы и маршрута в [MauiProgram.cs](MauiProgram.cs) и [AppShell.xaml](AppShell.xaml) (или текущей навигации через Shell).
- **Ресурсы:** Использовать существующие ключи из [AppResources.resx](Resources/AppResources.resx) (`RulesScoringTitle`, `PointsWinReg`, …); при необходимости добавить ключ для кнопки «Изменить» (например, `Edit` или `EditRules`).

Модель [TournamentRules](Domain/TournamentRules.cs) имеет только `init`-свойства, поэтому изменение возможно только через подстановку нового экземпляра: `Tournament.Rules = new TournamentRules { ... }`.

## 2. Большой вертикальный пробел в турнирной таблице

**Проблема:** Между строкой заголовков (# И В ВО ВБ …) и строками данных таблицы отображается большой пустой зазор.

**Причина:** В MAUI [CollectionView](Presentation/Views/TournamentDetailsPage.xaml) внутри [ScrollView](Presentation/Views/TournamentDetailsPage.xaml) получает неограниченную высоту; при такой раскладке он может «растягиваться» и вести себя как блок с лишним пустым пространством.

**Решение:**

- Для `CollectionView` с `ItemsSource="{Binding Standings}"` задать `VerticalOptions="Start"`, чтобы он занимал только высоту контента (строки данных), а не расширялся по вертикали.
- При необходимости заголовок и таблицу объединить в один блок с минимальным отступом: обернуть текущий `Grid` (заголовки) и этот `CollectionView` в общий контейнер (например, `VerticalStackLayout` с `Spacing="0"` или `Spacing="4"`), оставив основной `VerticalStackLayout` страницы с `Spacing="16"` между крупными секциями. Так зазор между заголовком и первой строкой данных станет предсказуемым и небольшим.

## 3. Отсутствие названий команд в турнирной таблице

**Проблема:** Во второй колонке таблицы (где ожидается название команды) ничего не отображается, видны только числа в других колонках.

**Причина:** В шаблоне элемента задано `ColumnDefinitions="30,*,40,..."`. В контексте [CollectionView](Presentation/Views/TournamentDetailsPage.xaml) внутри [ScrollView](Presentation/Views/TournamentDetailsPage.xaml) колонка `*` при измерении может получать нулевую или неверную ширину, из‑за чего `Label` с `TeamName` не отображается.

**Решение:**

- Заменить звёздочку на фиксированную минимальную ширину для колонки с названием команды в обоих местах — в строке заголовков и в [DataTemplate](Presentation/Views/TournamentDetailsPage.xaml) строки таблицы. Например, использовать `ColumnDefinitions="30,120,40,40,40,40,40,40,40,50"` (120 — ширина колонки «Команда»). Так колонка с `TeamName` всегда будет иметь ненулевую ширину и текст будет виден.
- При необходимости можно увеличить значение (например, до 140–150) для длинных названий; при этом длинные имена уже обрезаются за счёт `LineBreakMode="TailTruncation"` на [Label](Presentation/Views/TournamentDetailsPage.xaml).

Данные в [TournamentDetailsViewModel](Presentation/ViewModels/TournamentDetailsViewModel.cs) уже заполняются корректно (`TeamName = string.IsNullOrWhiteSpace(team.Name) ? "—" : team.Name`), поэтому достаточно исправить вёрстку.

## 4. Переименование пакета приложения

**Требование:** Заменить идентификатор пакета с `com.companyname.hockeytournamenttracker` на `com.iindi.hockeytournamenttracker`.

**Где меняется:**

- [HockeyTournamentTracker.csproj](HockeyTournamentTracker.csproj): в элементе `<ApplicationId>` установить значение `com.iindi.hockeytournamenttracker`. В MAUI этот идентификатор используется в том числе как package name для Android при сборке.
- В [AndroidManifest.xml](Platforms/Android/AndroidManifest.xml) пакет не задаётся явно (используется из проекта), дополнительных правок не требуется. При необходимости проверить, что после сборки Android-приложения в манифесте APK подставляется новый package.

Итого: одна правка в `.csproj`.
