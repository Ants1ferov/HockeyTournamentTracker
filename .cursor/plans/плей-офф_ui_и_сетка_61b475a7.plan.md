# Исправление экрана плей-офф

## Что проверено

- Основная разметка и кнопки находятся в [Presentation/Views/PlayoffBracketPage.xaml](Presentation/Views/PlayoffBracketPage.xaml).
- Логика построения геометрии сетки и загрузки раундов — в [Presentation/ViewModels/PlayoffBracketViewModel.cs](Presentation/ViewModels/PlayoffBracketViewModel.cs).
- Автозаполнение первого раунда по посеву вызывается только из [Presentation/Views/PlayoffBracketPage.xaml.cs](Presentation/Views/PlayoffBracketPage.xaml.cs).

## План изменений

- Исправить высоту и реактивность схемы сетки в [Presentation/ViewModels/PlayoffBracketViewModel.cs](Presentation/ViewModels/PlayoffBracketViewModel.cs):
  - сделать `CanvasHeight` уведомляемым свойством (`INotifyPropertyChanged`), чтобы `Grid HeightRequest` реально обновлялся после `BuildBracketGeometry()`;
  - при необходимости сделать уведомляемым `BracketBounds`, чтобы перестроение позиций карточек было стабильным при повторной загрузке/изменении настроек;
  - скорректировать верхний отступ геометрии (сейчас `topPadding = 22`) под заголовок раунда, чтобы карточки не заходили на бейдж раунда.
- Убрать автозаполнение первого раунда по посеву:
  - удалить/скрыть блок `Автозаполнение` (Picker + кнопка) в [Presentation/Views/PlayoffBracketPage.xaml](Presentation/Views/PlayoffBracketPage.xaml);
  - удалить обработчик `OnAutoFillSeedClicked` в [Presentation/Views/PlayoffBracketPage.xaml.cs];
  - зачистить неиспользуемые элементы VM, которые нужны только этому UI (например, `SeedSourceStages` и связанная загрузка) в [Presentation/ViewModels/PlayoffBracketViewModel.cs](Presentation/ViewModels/PlayoffBracketViewModel.cs), если после удаления UI они больше нигде не нужны.
- Улучшить отображение кнопок действий серии в [Presentation/Views/PlayoffBracketPage.xaml](Presentation/Views/PlayoffBracketPage.xaml):
  - заменить узкий `HorizontalStackLayout` для кнопок серии на более адаптивный контейнер (`FlexLayout` с переносом или двухрядный `Grid`), чтобы `Матчи серии` не обрезалась на узких экранах;
  - при необходимости слегка подправить `Padding/Spacing` кнопок в карточке серии для мобильной ширины.
- Исправить наложение текста в самой схеме в [Presentation/Views/PlayoffBracketPage.xaml](Presentation/Views/PlayoffBracketPage.xaml):
  - отделить заголовок раунда от области карточек (убрать отрицательный `Margin`, добавить безопасный верхний отступ);
  - для названий команд в карточках сетки задать аккуратные ограничения (`MaxLines`, `LineBreakMode`, размер/интерлиньяж), чтобы длинные имена не наезжали на соседние элементы.

## Проверка после изменений

- На стадии с 8 сериями первого раунда в блоке `Схема сетки` видны все серии и корректные соединители.
- На мобильной ширине кнопка `Матчи серии` читается полностью и не обрезается.
- Заголовки раундов и названия команд на схеме не перекрывают друг друга.
- Блок автозаполнения по посеву отсутствует, ручное редактирование серий работает как раньше.
