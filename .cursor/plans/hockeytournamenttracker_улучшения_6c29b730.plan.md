# План улучшений HockeyTournamentTracker

## Текущее состояние (кратко)

- **Турниры:** список на [MainPage.xaml](MainPage.xaml), переход по выбору в CollectionView, кнопка «Добавить»; удаления и смены статуса нет. Репозиторий уже имеет `DeleteAsync`.
- **Команды:** модель [Team](Domain/Team.cs) и [TeamRepository](Data/TeamRepository.cs) есть, но в UI команд нет — ни списка, ни добавления/редактирования. В форме матча [MatchEditPage.xaml](Presentation/Views/MatchEditPage.xaml) Picker берёт команды из `Teams`, поэтому без команд выбор пустой.
- **Матчи:** [Match](Domain/Match.cs) имеет `Status` (Scheduled, Finished, Cancelled); «Live» (матч идёт) не предусмотрен. В [TournamentDetailsViewModel](Presentation/ViewModels/TournamentDetailsViewModel.cs) матчи выводятся без разделения на текущие/завершённые и без отображения статуса.
- **Локализация:** строки захардкожены в XAML и коде, отдельной системы локализации нет.

## 1. Управление командами (добавление / удаление)

**Цель:** возможность добавлять и удалять команды турнира, чтобы в форме матча был выбор.

- **Экраны:**
  - **Список команд** — новая страница, открываемая из деталей турнира (кнопка «Команды»). Список команд турнира, кнопка «Добавить команду», у каждой команды — переход на редактирование и удаление (иконка корзины или меню «⋯»).
  - **Добавление/редактирование команды** — новая страница (TeamEditPage): название, краткое название, выбор иконки (п.2), сохранение через `ITeamRepository.SaveAsync`/`DeleteAsync`.
- **Навигация:** в [TournamentDetailsPage](Presentation/Views/TournamentDetailsPage.xaml) добавить кнопку «Команды» → переход на страницу списка команд с параметром `TournamentId`; из списка — переход на TeamEditPage (новый турнир: `TournamentId`, новый команда: пустой Id; редактирование: `TeamId`).
- **Shell:** зарегистрировать маршруты для страницы списка команд и TeamEditPage (по аналогии с [AppShell.xaml.cs](AppShell.xaml.cs)).

## 2. Иконки команд

**Цель:** у каждой команды — опциональная иконка (как в референсе Challenge Place).

- **Модель и БД:**
  - В [Team](Domain/Team.cs) и [TeamEntity](Data/Entities.cs) добавить поле `IconPath` (string, nullable) — путь к файлу иконки в локальном хранилище приложения (например `{LocalApplicationData}/TeamIcons/{tournamentId}_{teamId}.png`). При удалении команды — удалять файл.
  - В [TeamRepository](Data/TeamRepository.cs) маппинг Entity ↔ Domain расширить для `IconPath`. SQLite: при добавлении новой колонки нужна миграция — в [LocalDatabase](Data/LocalDatabase.cs) в `InitializeAsync` после `CreateTableAsync` выполнить проверку и при необходимости `ALTER TABLE Teams ADD COLUMN IconPath TEXT` (если колонки ещё нет), чтобы не ломать существующие установки.
- **UI:**
  - В TeamEditPage — выбор изображения: кнопка «Выбрать иконку» → `MediaPicker.PickPhotoAsync()` (или выбор из файла), сохранение файла в указанную папку, запись пути в `Team.IconPath`.
  - Отображение иконки: в списке команд — Image с привязкой к пути (конвертер или ImageSource из пути); в [MatchEditPage](Presentation/Views/MatchEditPage.xaml) в Picker — через ItemTemplate с Image + название команды (или отдельный кастомный контрол выбора команды с иконкой); при желании — в турнирной таблице и в списке матчей на [TournamentDetailsPage](Presentation/Views/TournamentDetailsPage.xaml) (иконка рядом с названием команды).

## 3. Удаление турниров

**Цель:** с главного экрана можно удалить турнир.

- В [MainPage.xaml](MainPage.xaml) в шаблоне карточки турнира добавить кнопку или иконку «⋯» / корзина (в углу карточки). По нажатию — либо контекстное меню («Удалить»), либо сразу `DisplayAlert` с подтверждением «Удалить турнир?» и при «Да» — вызов `ITournamentRepository.DeleteAsync(id)`, затем обновление списка (`LoadAsync`).
- Важно: не давать случайно нажать «удалить» вместо перехода в турнир — кнопка удаления отдельно от области перехода (см. п.4). Решение по каскаду: удалять связанные матчи и команды при удалении турнира (реализовать в репозитории или сервисе: сначала удалить матчи/команды по TournamentId, затем турнир) — иначе останутся «сироты» в БД.

## 4. Улучшение кликабельности карточки турнира

**Цель:** вся карточка турнира ведёт в детали турнира, без необходимости попадать в «узкий» пиксель.

- В [MainPage.xaml](MainPage.xaml): вместо (или вместе с) `SelectionChanged` у CollectionView сделать основной переход по **тапу по карточке**: на Frame (или внутренний Grid) повесить `TapGestureRecognizer`, в обработчике брать `tournament` из `BindingContext` элемента и выполнять `GoToAsync($"{nameof(TournamentDetailsPage)}?TournamentId={tournament.Id}")`. SelectionChanged можно оставить как запасной вариант или отключить, чтобы не было двойного срабатывания.
- Задать у карточки `MinimumHeightRequest` (например 56–64) и достаточный Padding, чтобы область нажатия была большой.
- Кнопку удаления (п.3) разместить так, чтобы её нажатие не считалось тапом по карточке (отдельный элемент с собственным GestureRecognizer или кнопка с `InputTransparent="False"` только на кнопке).

## 5. Статус турнира (Начался / Закончен и т.д.)

**Цель:** менять и сохранять статус турнира.

- В домене уже есть [TournamentStatus](Domain/Enums.cs): Planned, InProgress, Finished, Archived.
- В [TournamentDetailsPage](Presentation/Views/TournamentDetailsPage.xaml): добавить блок «Статус» с Picker или кнопками (Запланирован / В процессе / Завершён / В архиве). Привязка к `Tournament.Status` в ViewModel; при изменении — вызов `ITournamentRepository.SaveAsync(Tournament)` и при необходимости обновление отображаемых данных.
- [TournamentDetailsViewModel](Presentation/ViewModels/TournamentDetailsViewModel.cs): обеспечить возможность обновления статуса (команда/метод `UpdateStatus` или двухсторонняя привязка к свойству, которое при set вызывает SaveAsync).
- На [MainPage.xaml](MainPage.xaml) сейчас выводится `Text="{Binding Status}"` — enum отображается как «Planned» и т.д. Имеет смысл выводить русские подписи через конвертер или ресурсы (см. п.8).

## 6. Матчи в формате Live

**Цель:** показывать, что матч ещё идёт (не завершён), и визуально выделять такие матчи.

- **Модель:** в [Enums.cs](Domain/Enums.cs) добавить в `MatchStatus` значение `InProgress` (или `Live`). Обновить [MatchEntity](Data/Entities.cs) и [MatchRepository](Data/MatchRepository.cs) — маппинг по числу (Scheduled=0, InProgress=1, Finished=2, Cancelled=3). При необходимости миграция не нужна, если статус хранится как int и новые значения просто добавляются.
- **Логика:**
  - При создании матча статус по умолчанию — Scheduled.
  - «Начать матч»: установка статуса InProgress (отдельная кнопка в списке матчей или на экране матча).
  - «Завершить матч»: ввод счёта и типа исхода, установка статуса Finished (текущая форма сохранения матча уже выставляет Finished — оставить так для кнопки «Завершить»; для «идущего» матча можно открыть ту же MatchEditPage в режиме «завершить» с предзаполненными командами и датой).
- **UI на странице турнира:** в [TournamentDetailsViewModel](Presentation/ViewModels/TournamentDetailsViewModel.cs) формировать два списка (или один с группировкой): «Сейчас идут» (Status == InProgress) и «Расписание / Завершённые». В [TournamentDetailsPage.xaml](Presentation/Views/TournamentDetailsPage.xaml) отображать секцию «Сейчас идут» с бейджем «Live» (и время/арена при наличии), затем остальные матчи. У [MatchRow](Presentation/ViewModels/TournamentDetailsViewModel.cs) добавить свойство `IsLive` или `Status` и в шаблоне показывать индикатор (точка/бейдж «Live») для матчей InProgress.

## 7. Локализация (русский по умолчанию, задел под другие языки)

**Цель:** все строки выносить в ресурсы, по умолчанию русский, возможность позже добавить другие языки.

- Добавить в проект `Resources/AppResources.resx` (и при необходимости `AppResources.ru.resx` как default). В ресурсах — ключи для всех пользовательских строк: заголовки экранов, кнопки («Добавить», «Сохранить», «Команды», «Удалить», «Начать матч», «Завершить матч»), статусы турнира и матча, сообщения об ошибках и подтверждения.
- В XAML и коде заменить хардкод на привязку к ресурсам (например `x:Static` к ресурсам или через конвертер). Для enum (статусы турнира/матча) — конвертер или словарь в ресурсах (ключ по имени enum value).
- Структура: один .resx по умолчанию (русский); для расширения позже — добавление языковых папок/файлов (например `AppResources.en.resx`) и настройка платформы под смену языка. В плане ограничиться выносом строк в один файл ресурсов и использованием его везде.

## 8. Порядок реализации (рекомендуемый)

- Локализация (задел): создать AppResources.resx и заменить основные строки на главном экране и деталях турнира, чтобы дальше добавлять новые экраны уже с ресурсами.
- Команды: модель/БД (IconPath + миграция) → список команд и TeamEditPage → интеграция в детали турнира и в форму матча (выбор команды с иконкой).
- Турниры: удаление (кнопка + подтверждение + каскадное удаление в репозитории) и кликабельность карточки (TapGestureRecognizer + MinimumHeightRequest).
- Статус турнира: UI смены статуса на странице деталей турнира и отображение переведённого статуса на главной.
- Live-матчи: enum InProgress → секция «Сейчас идут» и бейдж Live в UI, кнопки «Начать матч» / «Завершить матч» при необходимости.

После этого приложение покроет: добавление/удаление команд и турниров, иконки команд, удобный переход в турнир, смену статуса турнира и отображение матчей в формате Live с заделом под локализацию.
