# План правок турнирной таблицы

## Текущее состояние (найденные причины)

- **П.1 — «Команда» сливается:** В [TournamentDetailsPage.xaml](Presentation/Views/TournamentDetailsPage.xaml) в строке заголовков два `Label` стоят в одной ячейке: `Grid.Column="2"` используется и для «Команда», и для «И» (Games). Они рисуются друг поверх друга.
- **П.2 — Группы:** ViewModel уже считает места по группам и заполняет `StandingRow.GroupName`, но в UI нет секций с заголовками групп (как в референсе Challenge Place). Столбец «Гр» выводится в таблице — его нужно убрать.
- **П.3 — Тексты столбцов:** Строки заданы в [AppResources.resx](Resources/AppResources.resx): «Заб», «Проп», «Посл. 5» — нужно заменить на «ГЗ», «ГП», «Результаты».
- **П.4 — Дублирование «+/-»:** В XAML заголовки в колонках 11 и 12 оба привязаны к `StandingsColGoalDiff`; в строках данных колонки 12 и 13 — одна для GoalDiff, вторая для Last5. То есть дублируется именно заголовок «+/-» (два столбца подряд с одной и той же подписью).
- **П.5 — Порядок и состав столбцов:** Сейчас порядок: #, Гр, Команда, (Games в данных в col 3), В, ВО, ВБ, П, ПО, ПБ, Заб, Проп, +/-, +/-, Посл.5, %, О. Столбец «И» в данных есть (`StandingRow.Games` и колонка 3 в строке), но в заголовках он «склеен» с «Команда» из‑за одинакового `Grid.Column="2"`. Нужный порядок: **#, Команда (с иконкой), О, И, В, ВО, ВБ, ПБ, ПО, П, ГЗ, ГП, +/-, Результаты, %** — и без столбца «Гр».
- **П.6 — Иконка команды:** В [Team.cs](Domain/Team.cs) есть `IconPath`. В [StandingRow](Presentation/ViewModels/TournamentDetailsViewModel.cs) поля для иконки нет; конвертер `PathToImageSource` уже есть и используется в [TeamsListPage.xaml](Presentation/Views/TeamsListPage.xaml).

## 1. Исправить слияние «Команда» и порядок столбцов (включая «О», «И»)

**Файл:** [Presentation/Views/TournamentDetailsPage.xaml](Presentation/Views/TournamentDetailsPage.xaml)

- Убрать столбец «Гр»: один столбец из `ColumnDefinitions`, один `Label` в заголовке и одна ячейка в шаблоне строки (сейчас колонки 1).
- Убрать дубликат «+/-»: оставить один заголовок и одну ячейку для разницы шайб.
- Задать порядок колонок по списку: **#, Команда, О, И, В, ВО, ВБ, ПБ, ПО, П, ГЗ, ГП, +/-, Результаты, %** (всего 15 колонок).
- В заголовках: каждой колонке — свой уникальный `Grid.Column` (0–14), без повторного использования одной и той же ячейки.
- В шаблоне строки: расставить все биндинги по тем же колонкам 0–14; колонку «Команда» оставить под индексом 1 (туда позже добавить иконку в п. 6).
- Обновить `ColumnDefinitions`: убрать ширину для «Гр», оставить одну ширину для «+/-», при необходимости скорректировать ширину колонки «Команда» под иконку + текст (например 100–120).

Итоговая сетка:

- 0: # (Place)
- 1: Команда (иконка + название)
- 2: О (Points)
- 3: И (Games)
- 4–9: В, ВО, ВБ, ПБ, ПО, П
- 10–11: ГЗ, ГП
- 12: +/-
- 13: Результаты (Last5)
- 14: %

## 2. Визуальное разделение по группам и удаление столбца «Гр»

**Логика отображения (как в референсе):** отдельные секции с заголовком группы и своей таблицей строк под ним.

**Вариант A (рекомендуемый):** одна таблица с групповыми заголовками.

- Ввести тип группы для отображения, например `StandingGroup` с полями `string GroupName` и `IList<StandingRow> Items`.
- Во ViewModel при наличии групп в правилах заполнять коллекцию групп (например `ObservableCollection<StandingGroup> StandingsByGroup`): каждая группа — название + список `StandingRow` для этой группы; команды без группы — одна секция «Без группы» или без названия.
- В XAML использовать `CollectionView` с `IsGrouped="True"`, `ItemsSource="{Binding StandingsByGroup}"`, задать путь к дочерним элементам группы (в MAUI обычно свойство вида `Items` или аналог) и задать `GroupHeaderTemplate`: заголовок с названием группы (крупный текст, как «Западная конференция» на референсе).
- Когда групп нет — использовать один «виртуальный» группу с пустым именем и всеми строками, чтобы один и тот же CollectionView работал и с группами, и без.

**Файлы:**

- [Presentation/ViewModels/TournamentDetailsViewModel.cs](Presentation/ViewModels/TournamentDetailsViewModel.cs): добавить тип `StandingGroup`, свойство `StandingsByGroup` (или аналог для группировки); при загрузке при наличии групп заполнять группы, иначе — один общий список. Таблицу строить без столбца «Гр» (поле `GroupName` в `StandingRow` можно оставить для группировки, но не выводить в ячейках).
- [Presentation/Views/TournamentDetailsPage.xaml](Presentation/Views/TournamentDetailsPage.xaml): перейти на привязку к группированному источнику и добавить `GroupHeaderTemplate` с заголовком группы. Убедиться, что в разметке строки таблицы колонки «Гр» нет.

**Вариант B (проще, без IsGrouped):** оставить плоский `Standings`, но перед блоком таблицы выводить несколько блоков: для каждой группы — `Label` (имя группы) + тот же по структуре вывод строк (например, вложенный `CollectionView` с `ItemsSource` по отфильтрованному списку). Это потребует либо нескольких коллекций во ViewModel (по одной на группу), либо фильтра по группе в конвертере/коде, и усложнит привязку. Вариант A с одной группированной структурой проще поддерживать.

Конвертер `PathToImageSource` уже зарегистрирован в [App.xaml](App.xaml); при отсутствии иконки `Image` может быть пустым (конвертер возвращает null).

- Обновить отображаемые подписи под формулировки пользователя:
  - `SortCriterionWinsOvertime`: заменить текущее значение на **«Победы в овертаймах»** (сейчас «Победы в овертайме»).
  - `SortCriterionWinsShootout`: заменить текущее значение на **«Победы в буллитах»** (сейчас «Победы по буллитам»).

После этого в UI редактирования правил и в отображении порядка распределения мест на экране деталей турнира будут показываться эти названия, и оба критерия по-прежнему доступны в списке порядка сортировки.
