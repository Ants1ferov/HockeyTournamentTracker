# План: иконки вкладок, удаление стадий, таблица по стадиям

## 1. Иконки и подписи на панели вкладок

**Текущее состояние:** нижняя панель в [TournamentDetailsPage.xaml](Presentation/Views/TournamentDetailsPage.xaml) (строки 345–357) — три `Button` только с текстом.

**Иконки:** использовать файлы из `Icons/` (или скопировать в [Resources/Images](Resources/Images), т.к. в проекте уже есть `MauiImage Include="Resources\Images\*"`). Нужны 6 иконок вкладок и 1 для удаления:

- Главная: `tourHomeNonClicked.png` / `tourHomeClicked.png`
- Стадии: `tourStageNonClicked.png` / `tourStageClicked.png`
- Участники: `tourPlayerNonClicked.png` / `tourPlayerClicked.png`
- Удаление стадии: `iconDeleteTournament.png` (в Resources/Images уже есть `icondeletetournament.png` — проверить имя).

**Шаги:**

- Скопировать перечисленные PNG из папки `Icons` (или из `assets/`, куда сохранены приложенные файлы) в [Resources/Images](Resources/Images), чтобы они подхватывались как MauiImage (алиас = имя файла без расширения).
- В [TournamentDetailsViewModel.cs](Presentation/ViewModels/TournamentDetailsViewModel.cs) добавить свойства только для отображения иконки вкладки (строка-алиас):
  - `HomeTabIcon` → при `IsHomeTabSelected` возвращать `"tourhomeclicked"`, иначе `"tourhomenonclicked"`.
  - `StagesTabIcon`, `ParticipantsTabIcon` — по той же логике для стадий и участников.
  - В сеттере `SelectedTabIndex` вызывать `OnPropertyChanged` для этих трёх свойств.
- В XAML заменить каждую кнопку вкладки на вариант с иконкой сверху и текстом снизу:
  - Использовать `Button` с `ImageSource="{Binding HomeTabIcon}"` и `Text="{x:Static res:AppResources.TabHome}"`, свойство `ContentLayout="Top,4"` (или аналог в MAUI: верхнее положение изображения относительно текста с отступом).
  - Аналогично для «Стадии» и «Участники» с привязкой к `StagesTabIcon` и `ParticipantsTabIcon`.
- Если привязка `ImageSource` к строке не сработает (алиас MauiImage), добавить конвертер «string → ImageSource» по аналогии с [PathToImageSourceConverter](Presentation/Converters/PathToImageSourceConverter.cs) и привязывать через него (например, для embedded: использовать принятый в проекте способ получения ImageSource по имени ресурса/алиасу).

**Итог:** у каждой вкладки — иконка (меняется при выборе) и подпись внизу («Главная», «Стадии», «Участники»).

## 3. Удаление стадий с иконкой

**ViewModel:** в [TournamentDetailsViewModel.cs](Presentation/ViewModels/TournamentDetailsViewModel.cs) добавить метод `DeleteStageAsync(Guid stageId)`:

- проверка `Tournament != null`;
- вызов `_stageRepository.DeleteAsync(stageId)`;
- если `SelectedStage?.Id == stageId`, сбросить `SelectedStage = null`;
- вызов `LoadAsync(Tournament.Id)` для обновления списка стадий и данных.

**Страница:** в шаблоне элемента стадии (вкладка «Стадии», [TournamentDetailsPage.xaml](Presentation/Views/TournamentDetailsPage.xaml)) в карточку стадии добавить кнопку/иконку удаления:

- в `Grid` с колонками `*,Auto` во вторую колонку поместить `ImageButton` или `Button` с `ImageSource` (алиас для `iconDeleteTournament.png` / `icondeletetournament.png`) и `Clicked="OnDeleteStageClicked"`.
- в обработчике `OnDeleteStageClicked` в [TournamentDetailsPage.xaml.cs](Presentation/Views/TournamentDetailsPage.xaml.cs) из контекста получить `Stage` (например, через `sender` и родительский `BindingContext`), при необходимости показать подтверждение («Удалить стадию?»), затем вызвать `_viewModel.DeleteStageAsync(stage.Id)`.

**Ресурсы:** при необходимости добавить строку «Удалить стадию?» в [Resources/AppResources.resx](Resources/AppResources.resx) и использовать в диалоге подтверждения.

## 4. Турнирная таблица только у стадии; кликабельные стадии

**Текущее состояние:** таблица (StandingsByGroup) строится в `LoadAsync` по всем матчам турнира и показывается на главной (строки 104–176 в TournamentDetailsPage.xaml).

**Требование:** таблицу убрать с главной; отображение турнирной таблицы — только при выбранной стадии, причём таблица считается только по матчам этой стадии.

**ViewModel:**

- Добавить коллекцию для таблицы выбранной стадии, например `ObservableCollection<StandingGroup> StandingsByGroupForStage`.
- В `RefreshStageMatches()` (или в сеттере `SelectedStage`) после получения `stageMatches` (матчи выбранной стадии):
  - вызвать `_statsService.CalculateStandings(Tournament, teams, stageMatches)` (только матчи стадии);
  - по тому же алгоритму, что и в `LoadAsync`, построить группы строк `StandingRow` (с учётом групп из правил турнира и `GetLast5ResultsForTeam` по `stageMatches`), заполнить `StandingsByGroupForStage` и вызвать `OnPropertyChanged` на основном потоке (как для `StageMatches`).
- На главной больше не заполнять и не использовать `StandingsByGroup` для отображения (можно не заполнять его в `LoadAsync` или оставить пустым для совместимости, если где-то ещё привязан).

**XAML:**

- На вкладке «Главная» удалить весь блок с заголовком «Турнирная таблица» и `ScrollView`/`CollectionView` с `StandingsByGroup` (строки 104–176).
- На вкладке «Стадии» при выбранной стадии (`SelectedStage != null`) показывать:
  - заголовок «Турнирная таблица» (или «Таблица стадии»);
  - тот же по структуре блок таблицы (горизонтальный ScrollView + Grid заголовков + CollectionView с `StandingsByGroupForStage`, те же колонки и шаблоны строк), привязанный к `StandingsByGroupForStage`;
  - ниже — уже существующие блоки «Матчи» и «Добавить матч» со списком `StageMatches`.

Стадии уже кликабельные (выбор через `SelectionChanged` и `SelectedStage`); при выборе стадии отображаются таблица этой стадии и матчи — дополнительных изменений логики выбора не требуется.
